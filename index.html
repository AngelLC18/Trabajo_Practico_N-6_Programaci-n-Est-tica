1) Las versiones de JavaScript que tenemos en el mercado son las siguientes:

Ver Nombre Descripción
ES1 ECMAScript 1 (1997) Primera edicion
ES2 ECMAScript 2 (1998) Cambios de editorial
ES3 ECMAScript 3 (1999) Se añade expresiones regulares: try/catch, switch,do-while
ES4 ECMAScript 4 Nuncas se lanzo
ES5 ECMAScript 5 (2009) Añede el "strict mode ", JSON support, String.trim(), Array.isArray(), Metodos de iteración de arrelgos permitiendo comas finales para objetos literales
ES6 ECMAScript 6 let y const para declarar variables y arrow functions. Let permite la reasignación de valores de una variable. Con const el valor no puede ser reasignado. Ambas tienen un scope de bloque. Untitled Arrow functions Es una alternativa compacta para declarar una function.
ES7 ECMAScript 7 Exponentiation operator Se introdujo una alternativa a Math.pow para poder realizar operaciones más claras.
ES8 ECMAScript 8 Object entries Es un función que no permite transformar un objeto en un array con cada una de las propiedades en un array de la forma [key, value]
ES9 ECMAScript 9 Expresiones regulares En esta versión de ECMA se incorporó la posibilidad de usar grupos nombrados usando una expresión regular. Un match de una expresión regular retorna un objeto. Si un fragmento de la expresión regular esta dentro de los paréntesis, es capturado por el grupo y es almacenado en el grupo en el cual hizo el match.
ES10 ECMAScript 10 lat El método flat de los array permite aplanar los elementos internos de un array permitiendo eliminar array internos.
ES11 ECMAScript 11 Optional chaining Es una forma de evitar errores solicitando propiedades de forma opcional utilizando el símbolo ? , sirve cuando no estamos seguros de que los valores estén completos. Este operador es igual que . , excepto que en vez de causar un error si una referencia es nullish(null or undefined), la expresión retorna un valor undefined.
ES12 ECMAScript 12 Numeric-separators Nos permite separar los números con _ con la finalidad que la lectura del número sea más fácil y clara. const number = 1_000_000_000 console.log(number) // 1000000000 ReplaceAll Retorna un nuevo string con todas las coincidencias de una patrón remplazado por la propiedad replacement.
ES13 ECMAScript 13 At El método at() recibe un valor numérico entero y devuelve el elemento en esa posición, permitiendo valores positivos y negativos. Los valores negativos contarán desde el último elemento del array.
